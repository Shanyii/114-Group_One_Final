# 期末專題主題提案報告

# StudyAgent AI - 課堂講義學習與複習規劃系統

協助學生智慧解析課堂講義、自動整理重點、生成測驗並依學習狀況動態規劃個人化複習流程的 AI Agent 系統。

---

## 1. 專題名稱與一句話說明

* **專題名稱**：StudyAgent AI — 課堂講義學習與複習規劃系統
* **系統類型**：AI 學習輔助與個人化教學 Agent 系統
* **目標任務**：協助學生在課後上傳 PDF/PPT 課堂講義，自動進行智慧摘要、客製化出題、答題批改，並根據學生弱點與考試日期動態產生個人化複習計畫。
* **使用者成果**：使用者可獲得一份講義重點摘要與專有名詞對照表、一組針對個人弱點設計的實戰測驗題、詳細的答錯觀念解析，以及動態調整的個人化複習時間表。

---

## 2. 目標使用者與痛點

### 2.1 目標使用者
* **主要使用者**：大學生及自主學習者（需要面對大量課堂投影片、PDF 講義，並準備期中/期末考試的學生）。
* **使用情境**：在期中/期末考前 1-2 週複習課堂講義，或者在每週課後進行自主學習評估時。

### 2.2 痛點分析
1. **講義篇幅龐大，重點分散**：課堂 PPT 往往有數十甚至上百頁，充斥圖表與破碎文字，學生難以在短時間內提取核心觀念。
2. **缺乏客觀評估，不知理解程度**：僅看講義容易產生「我已經懂了」的錯覺，缺少即時且客觀的自我測驗機制。
3. **無效複習，時間分配不均**：學生不知道自己到底卡在哪個特定知識點（例如不知道自己是「梯度下降」觀念懂了，但「L2 正規化」公式算錯），導致平均分配時間，複習效率低。
4. **固定流程工具無法適應個人差異**：市面上一般的學習網站或 FAQ 只能提供固定的題目或答案，無法根據學生多次作答的歷史錯誤數據，動態調整推薦主題與時程。

### 2.3 為什麼需要 AI Agent？
一般的靜態網站或固定 FAQ 只能做關鍵字檢索，無法「理解」講義語意，更無法執行複雜的分步任務。
本系統需要 AI Agent 來：
1. **動態分流與任務編排**：後端預留意圖判斷介面，前端在點擊啟動分析後，自動指派完整流程任務（同時進行摘要濃縮與測驗題目生成）。
2. **RAG 知識調回與推理**：從海量講義文字中，檢索最相關的知識片段，避免 AI 憑空捏造（Hallucination）。
3. **狀態更新與自我優化 (Memory Update)**：Agent 會將學生每次的答題結果加權記錄至 SQLite 資料庫，形成「弱點記憶庫」，並在下次生成題目或計畫時，自動將弱點權重調高，實現真正在線個人化教學。

## 3. 核心任務流程

### 3.1 核心任務問答說明
* **使用者輸入**：上傳 PDF / PPTX 講義檔案（或直接貼上講義純文字），並點擊「啟動 AI Agent 分析」。
* **Agent 先判斷什麼**：後端會先由 `Intent Agent` 解析輸入內容（若為自訂文字，則進行主題分類與任務類型判斷；若為上傳講義，則直接由前端預設請求觸發「摘要與出題」的完整工作流）。
* **Agent 會查詢什麼資料**：RAG 模組會使用 `text-embedding-004` 模型，在本地 `ChromaDB` 持久化向量庫中查詢與講義主題最相關的段落，作為後續摘要與出題的上下文依據。
* **Agent 會呼叫哪些工具**：
  * `DocumentParser`：解析並進行智慧遞迴切分。
  * `RAGRetriever`：檢索相關段落。
  * `LLMClient`：呼叫 LLM 進行摘要、出題、批改與計畫生成。
  * `StateRepository` 與 `LogRepository`：讀寫 SQLite 資料庫以儲存狀態與任務 Log。
* **Agent 如何更新狀態**：當學生提交答案後，系統會自動批改，並在 `students` 資料表中更新該學生的 `weak_topics`（對應主題錯題次數 +1），並記錄已完成的章節。
* **最後輸出成果**：
  1. 結構化講義重點摘要與詞彙表。
  2. 動態測驗題卡與作答即時批改解析。
  3. 依據弱點權重與考試時間排定的個人化複習計畫。
  4. 即時顯示在前端的 Agent 工作歷程日誌 (SSE Stream)。

## 4. Agent 架構設計

本系統架構採分工明確的協同 Agent/模組設計，各模組職責如下：

| 模組 / Agent | 任務 | 輸入 | 輸出 |
| :--- | :--- | :--- | :--- |
| **Intent Agent** | 語意分析使用者指令，識別具體意圖（摘要/出題/規劃/完整） | 使用者自然語言指令 | 任務意圖型態 (Intent Type) |
| **Document Parser** | 提取上傳的 PDF/PPT 文字，並按投影片/頁碼智慧切分 | 原始講義檔案 (Binary) | 帶有頁碼資訊的文字 Chunks 陣列 |
| **RAG Retriever** | 基於關鍵字與語意搜尋，從講義向量索引中召回相關內容 | 查詢關鍵字或題目主題 | 最相關的講義段落 (Passages) 陣列 |
| **Summarizer Agent** | 彙整講義核心內容，以結構化 Markdown 輸出重點整理與詞彙表 | 講義文字 Chunks | 重點摘要與核心詞彙對照表 |
| **Quiz Generator** | 根據講義內容與學生的弱點，動態生成客製化的測驗題目 | 主題與測驗題數 | 選擇題/是非題/簡答題 JSON 列表 |
| **Grader Agent** | 對學生的答案進行語意批改，並提供詳細的對錯解析與觀念補充 | 題目、學生答案、正確答案 | 批改狀態 (對/錯)、解析說明文字 |
| **State Updater** | 更新學生學習記憶（弱點計數加權、已完成章節等） | 批改結果與主題標籤 | 更新後的學生學習狀態資料 (SQLite) |
| **Plan Generator** | 根據學生最新的弱點主題與考試時間，生成複習時程計畫 | 弱點主題、天數或考試日期 | 個人化複習時間表與行動建議 |

## 5. RAG / 資料來源設計

為了防止大語言模型憑空捏造（幻覺），本系統的核心知識均來自於使用者上傳的真實課堂教材。

* **資料來源**：學生自行上傳的 PDF 講義或 PPT/PPTX 投影片檔案。
* **資料格式**：解析後的純文字 (Text)，切分為段落後轉為 Embedding 向量存入 `ChromaDB`，並將解析純文字暫存於 SQLite 的 `documents` 資料表。
* **資料建立方式**：後端使用 `pdfplumber` 解析 PDF，`python-pptx` 解析 PPTX，獲取包含頁碼資訊的原始文字，並呼叫 `chunk_document` 完成智慧段落切分。
* **查詢方式**：混合檢索模式。透過 Google `text-embedding-004` 模型將使用者查詢轉為 768 維向量，於 ChromaDB 中進行餘弦相似度檢索（Cosine Similarity Search），找出與意圖最相關的前 3-5 個 Chunks。
* **使用時機**：
  1. 當 Agent 判斷需要「整理重點」時，會調用全文 Chunks 進行摘要。
  2. 當 Agent 需要針對特定主題「生成題目」時，會檢索與該主題最相關的段落作為 LLM 的 Context。
* **來源標示**：生成重點摘要與測驗題目時，系統會在 JSON Response 中標註資料來源的頁碼（例如：「來源：投影片第 4 頁」），前端會將此頁碼標註於 UI 上，方便學生對照原始教材。
* **如何避免憑空生成**：在 System Prompt 中加入嚴格防禦條款：*「你只能基於給定的 Context（講義內容）進行摘要與出題。如果 Context 中沒有相關知識，請回答不知道，嚴禁胡亂捏造任何講義中未提及的公式、專有名詞與概念。」*

## 6. Tool Calling 設計

本系統規劃了 8 個核心工具函式，用以協調各個 Agent 完成任務：

| 工具名稱 | 功能 | 輸入 | 輸出 | 觸發時機 |
| :--- | :--- | :--- | :--- | :--- |
| **chunk_document** | 將講義原文依頁數與長度智慧切分為段落 | 原始文字字串 | `paragraphs[]` | 講義檔案解析完成後 |
| **retrieve_content** | 執行 RAG 語意檢索，召回相關段落 | 關鍵字或問題主題 | `relevant_paragraphs[]` | 執行摘要或出題前 |
| **generate_summary** | 調用 LLM 彙整段落內容，產出結構化重點 | 講義段落列表 | `summary_markdown` | Intent 包含 `summary` 時 |
| **generate_quiz** | 調用 LLM 根據講義內容與弱點生成考題 | 檢索段落、題數、偏好題型 | `quizzes_json` | Intent 包含 `quiz` 時 |
| **grade_answer** | 批改學生提交的答案並提供個人化解析 | 題目、學生答案、正確答案 | `{is_correct, explanation}` | 學生提交測驗作答時 |
| **update_learning_state**| 根據測驗結果更新學生的錯題弱點統計 | 答題結果、學生 ID | `updated_learning_state` | 批改完答案後 |
| **generate_study_plan** | 根據學生的弱點記憶與考試時間規劃複習時程 | 學生弱點主題、考試日期 | `study_plan_markdown` | 任務意圖包含 `plan` 時 |
| **save_log** | 持久化本次任務的完整執行日誌 (技術/可讀) | 任務資訊、工具呼叫鏈 | `log_id` | 任務鏈執行完畢或失敗時 |

## 7. 預期畫面與操作方式

本系統規劃了三個核心操作頁面，確保流暢的學習體驗：

### 7.1 首頁：講義上傳與範例選擇頁
* **介面元素**：拖曳式上傳區域（支援 PDF / PPTX / TXT / MD）、自訂簡報文字貼上區（Textarea）、快速測試內建講義範例卡片。
* **操作流程**：學生拖曳上傳講義檔案，或者貼上課程純文字（也可以直接點選右側的內建範例），接著點擊「啟動 AI Agent 分析」按鈕，系統即跳轉至工作頁。

### 7.2 工作頁：Agent 工作歷程控制台
* **介面元素**：左側為「Agent 終端日誌 (Agent Terminal)」，右側為「動態進度條與狀態指示器」。
* **操作流程**：後端透過 Server-Sent Events (SSE) 即時將 Agent 執行的每一步驟推送至前端。終端會以打字機效果即時渲染：「`[Step 1] 解析講義中...`」、「`[Step 2] RAG 查詢完成`」。學生能清楚看見 Agent 當前的思考脈絡，消除等待焦慮。

### 7.3 結果頁：學習回饋與 Dashboard 看板
* **介面元素**：
  * **重點整理面板**：以卡片化呈現 Markdown 重點摘要與名詞解釋，標註出處頁碼。
  * **測驗互動卡片**：展示 Agent 生成的測驗，學生可點選作答，送出後即時在卡片下方顯示 Grader 的批改對錯與名詞詳細解析。
  * **弱點分析圖表**：使用 Chart.js 展示該學生在資料庫中累積的常錯主題（如「過擬合」錯 3 次，「梯度下降」錯 1 次）。
  * **複習計畫表**：以日曆或清單形式呈現 Plan Generator 生產的複習時間規劃。

## 8. 期末 Demo 情境設計

### 8.1 Demo 流程規劃
1. **登入與初始化**：展示註冊與登入介面，登入帳號 `qqq`。此時展示弱點 Dashboard，顯示所有數值皆為初始狀態。
2. **上傳與啟動分析**：上傳一份「**專案版控與協作**」的講義 PDF。確認上傳成功後，直接點擊「啟動 AI Agent 分析」。
3. **即時執行展示**：畫面跳轉至工作頁，台下觀眾與評審能看到 Agent Terminal 即時跳出步驟，從 `read_document` 直至 `generate_quiz` 完成。
4. **重點摘要展示**：展示生成出的重點摘要，內容涵蓋 Git 基本指令、分支策略、Pull Request 流程、Merge Conflict 處理等核心知識點，名詞解釋中清晰標註來源頁碼。
5. **作答與批改**：學生在畫面上對產生的選擇題進行作答（例如「什麼情況下會發生 Merge Conflict？」）。提交後，畫面上即時揭示 Grader 的解析。
6. **記憶更新與計畫產生**：故意答錯題目。此時，弱點 Dashboard 中對應的知識點（如「Merge Conflict 解法」）錯誤次數從 0 變為 1。同時，系統動態產生個人化複習計畫，優先複習版控協作的弱點主題。

| 測試案例 / 面向 | 測試操作步驟 | 預期結果 | 達成 |
| :--- | :--- | :--- | :---: |
| **1. 講義上傳與解析完整性** | 上傳包含圖表、表格與中英混合內容的「專案版控與協作」PDF 講義。 | 系統能正確提取文字內容，並在合理時間內完成全文切分與 ChromaDB 向量索引建置，不遺漏關鍵段落。 | ✅ |
| **2. 重點摘要與詞彙閃卡品質** | 完成講義上傳後，觀察 AI 自動生成的重點摘要卡片與核心詞彙閃卡內容。 | 摘要內容應涵蓋講義的主要知識點（如 Git 分支策略、PR 流程），詞彙閃卡的定義應與講義原文吻合，無明顯捏造或偏離。 | ✅ |
| **3. 測驗出題與批改正確性** | 完成摘要後切換至測驗頁，進行作答並提交。 | 題目應與講義內容直接相關；提交後系統能正確判斷對錯，並針對答錯題目提供 AI 詳細解析說明。 | ✅ |
| **4. 弱點記憶持久度（跨 session）** | 註冊並登入帳號後進行測驗，故意答錯特定題目，隨後重新整理頁面並再次登入。 | 重新登入後，學習歷程中該題目的答錯紀錄依然存在，確認 SQLite 持久化儲存正常運作。 | ✅ |
| **5. AI 隨身問答準確度（RAG）** | 上傳講義後，切換至「講義 AI 問答」頁籤，針對講義內容提問（如「什麼是 Merge Conflict？」），再詢問一個講義中未提及的主題。 | 針對講義內容的提問，系統應從向量庫召回相關段落並生成準確回答；對於講義未提及的主題，系統應提示無相關內容，不憑空捏造。 | ❌ |
